What is Mutch?

The professional hiring platform built on mutual intent

The Problem

LinkedIn is a résumé database with spam. Candidates apply into black holes. Recruiters blast 500 InMails. No mutual intent signal exists anywhere in the current hiring stack.

The Solution

Swipe-based mutual interest matching for professionals. A match only happens when both sides want it. Dramatically higher conversion from first contact to hire.

The Market

Global recruitment market is $650B+. India alone has 560M working-age people, 50M+ active job seekers. No swipe-based hiring app has achieved mass adoption in India yet.

Core Proposition

Mutch is not a job board. It is not a social network. It is a mutual-intent hiring marketplace — the first platform where both the employer and the candidate must express interest before any conversation begins. This single mechanic eliminates 90% of hiring friction on both sides.

Implementation Phases

Step-by-step build plan from MVP to scaled platform

● Phase 1

Core MVP — Weeks 1–8

1
Authentication System

OAuth (Google, LinkedIn) + Email/Password with JWT + Refresh Token rotation. Existing code can be reused here. Implement device fingerprinting from day one for fraud prevention.

Next-Auth / Passport.js JWT + Redis Google OAuth 2.0 LinkedIn OAuth
2
Onboarding Flow

Multi-step onboarding with LinkedIn profile import via API. Pull: name, headline, experience, skills, education, location. Store enriched profile in Postgres with JSONB for flexible skill tagging. Target: under 3 minutes to first swipe.

LinkedIn API v2 Postgres + JSONB React Hook Form Zod validation
3
Role Selection & Profile Builder

After basic onboarding, user selects: Employer or Employee (can hold both modes — founders often do). Employer fills: company, role, CTC range, perks, JD (500 chars). Employee fills: desired role, expected CTC, skills, preferred companies. Both profiles stored separately; a single user account can have both.

Postgres multi-profile schema S3 (logo/photo) Cloudinary
4
Swipe Deck Engine

Core swiping UI. Cards rendered from a pre-fetched queue (batch of 20, prefetch next batch at 5 remaining). Swipe data logged to a swipes table with timestamp, action (left/right/superswipe), session_id. Card shows: photo, name, title, 3 skills, exp years, location, CTC range. No CV shown here — this is intentional and protects the premium unlock mechanic.

React Native Reanimated Gesture Handler Redis queue Postgres swipes table
5
Match Detection & Mutual Match Flow

When user A right-swipes user B: check if B already right-swiped A. If yes → create Match record, trigger match animation, unlock chat. Match table: match_id, user_a, user_b, created_at, status (active/archived/reported). Run match check via Redis pub/sub for real-time detection without polling Postgres on every swipe.

Redis Pub/Sub Postgres matches WebSocket
6
Basic Messaging (Chat)

Chat unlocked only on mutual match. Opening screen shows 5 pre-written starter prompts (e.g. "I'd love to know more about this role", "What's the interview process like?") plus a free-text input. Share CV button appears in chat tray — sends CV as a document message. Messages stored with end-to-end encryption flag. Build on WebSocket for real-time delivery.

Socket.io Postgres messages AES-256 encryption S3 (file share)
● Phase 2

Intelligence Layer — Weeks 9–16

7
Smart Filter Engine

Employer filters: company size, industry, role type (FT/freelance/contract), CTC offered range, required exp, skills (multi-select with autocomplete), location (city/remote/hybrid), qualifications, notice period tolerance. Employee filters: company type (startup/MNC/FAANG), WFH preference, CTC expected range, role, growth stage, team size. Filters stored per-user; remembered across sessions. Filter quality directly impacts match quality — this is a core differentiator.

Elasticsearch Postgres full-text Redis filter cache
8
AI Compatibility Score (Proprietary)

Hidden backend score (0–100) computed for every employer-employee pair using: skill overlap, CTC alignment, location match, industry history, seniority fit, and behavioural signals (time spent on card, re-views, swipe speed). This score determines card ordering in the swipe deck — highest compatibility appears first. Users never see the raw score; they only experience better matches. This is core IP.

Python ML service scikit-learn / XGBoost FastAPI pgvector (embeddings)
9
Profile Verification System

LinkedIn badge (OAuth confirmed), Email verification, Phone OTP. For employers: optional company domain email verification (hiring@company.com = verified badge). For employees: optional document upload (offer letter, degree) for "Verified Experience" badge. Badges displayed on cards — strong trust signal that improves swipe rates.

Twilio OTP SendGrid AWS Textract (doc verify)
10
Reporting, Blocking & Safety Layer

Report user (fake profile, harassment, misleading info), Block user (immediate removal from each other's decks), Rate employer after conversation (1–5 stars, feeds into Employer Rating shown on card). All reports go to admin queue with auto-severity tagging via keyword classifiers. Repeated flags trigger account suspension workflow.

OpenAI Moderation API Admin queue (Postgres) Block list (Redis set)

Complete Feature Set

Every feature the platform will support

Employer Features

Job Post Creation

Company name, logo, role title, employment type (full-time / part-time / contract / freelance / internship), CTC range (₹ or $), job location, remote/hybrid toggle, required experience, qualifications, skills (tag-based), perks (ESOP, WFH, health insurance, etc.), short JD (500 chars), tech stack tags.

Multiple Job Posts

Employers can maintain up to 5 active job posts simultaneously (free tier). Each post has its own swipe deck, filter settings, and match list. Premium removes the cap.

Employer Filters (Swipe Deck)

Filter employee deck by: skills (multi), experience range, expected CTC range, current location, willing to relocate, notice period, employment history, education level, language, availability (immediate / 1 month / 2 months+).

Candidate Pipeline View

All matches visible in a Kanban-style pipeline: Matched → Messaged → Interview Scheduled → Offer Sent → Hired / Rejected. Status updated manually or via in-app triggers.

Super Swipe (Premium)

Sends candidate an immediate push notification: "An employer is very interested in your profile." 3x match rate vs standard right swipe. 5 super swipes/week on premium plan.

Who Liked You (Premium)

See blurred cards of employees who already right-swiped on the job post. Swipe right to match instantly — no waiting. Highest conversion feature, kept behind premium paywall.

Team Hiring (Seat-based)

Add hiring managers and sourcers under one company account. Each seat can manage their own job posts. HR admin sees unified pipeline across all posts. RBAC: Admin, Recruiter, Viewer roles.

Analytics Dashboard

Card impressions, right-swipe rate, match rate, response rate, avg time-to-match, conversion funnel (match → hired). Helps employers understand if their job card is compelling or needs editing.

Employee Features

Employee Profile

Photo, name, current role, current company, total experience, skills (tag-based), preferred role, expected CTC, preferred location, availability, education, brief bio (160 chars), portfolio link, GitHub/Behance, LinkedIn URL.

Employee Filters (Swipe Deck)

Filter employer/job deck by: company type (startup/MNC/FAANG/Series-A-B/bootstrapped), industry, CTC offered range, remote/hybrid/onsite, job location, perks (ESOP must-have toggle), company rating (Mutch-native + Glassdoor), growth stage, team size.

Stealth Mode

Hide profile from employers whose domain matches current employer's email domain. Allows safe passive job seeking. Premium feature — high perceived value for employed candidates.

Profile Boost (Premium)

Profile appears 2–3x more in employer swipe decks for 7 days. Shown with a subtle "Highlighted" badge. Can be targeted by job type or company type for precision boosting. Sold as one-time purchase or subscription add-on.

Profile Insights (Premium)

See: how many employers viewed your card, how many right-swiped, which industry/company types are interested, profile completion score with actionable tips. Strong retention hook — employees log in daily to check numbers.

AI Profile Coach (Premium)

GPT-powered analysis of profile vs. filter criteria of companies the employee is targeting. Suggests: which skills to add, how to reword bio, what CTC range is competitive, whether to add portfolio. Personalized, data-driven advice.

Company Ratings

After any conversation or interview, employee can rate the employer: communication quality, process transparency, offer fairness, culture impression. Ratings aggregated to company card — visible to all employees. Authentic data creates a flywheel: better companies get more swipes.

CV Builder (Optional)

Generate a clean, formatted CV from Mutch profile data. Available in-app as a PDF download or as the "Share CV" action in chat. Means even users without a polished CV can participate. CV is never shown during swiping.

Matching Engine (Proprietary)

The matching engine is the core IP. It determines which profiles appear in whose deck, in what order, and how to improve match quality over time. Never expose the algorithm — it is your primary competitive moat.

Compatibility Score (0–100)

Computed server-side per pair. Inputs: skill vector similarity (cosine), CTC overlap %, location match, experience bracket match, industry history, behavioural signals. Score determines stack rank in swipe deck — never shown to user.

Behavioural Signal Learning

Track: time spent on each card, scroll depth (if profile expanded), re-views, swipe velocity, message response time. Feed into compatibility model as implicit signals. Improves recommendations over time without explicit feedback from users.

Elo-Style Reputation System

Each profile has a hidden "desirability score" updated after every swipe interaction (right swipes increase, left swipes decrease, weighted by swiper's own desirability). Higher-rated profiles shown first in decks of users at similar rating tier — prevents top profiles from being shown to everyone and burning out.

Freshness & Activity Weighting

Recently active profiles boosted in deck (logged in within 48h = +15% deck priority). Job posts marked "urgent" get temporary boost. Prevents stale profiles from clogging decks and wasting swipes.

Hard Filter → Soft Rank

Filters are applied in two stages: hard filters remove ineligible profiles (e.g. CTC completely out of range), then soft ranking orders the remainder by compatibility score. This ensures filter precision while using AI to rank within results.

Diversity Injection

5–10% of deck is "wildcard" profiles slightly outside the main compatibility range. Prevents filter bubbles and exposes users to serendipitous matches they wouldn't have sought — increases session length and discovery.

Messaging System

Starter Prompt Library

5 curated openers shown at match screen. Employer starters: "Tell me about your recent projects", "What's your ideal work environment?". Employee starters: "What does the interview process look like?", "Is the CTC range negotiable?". Reduces cold-start awkwardness — key to first-message conversion.

Share CV Action

Employee can send CV as an in-chat document at any point. Employer can send Job Description PDF. Files encrypted at rest, downloadable for 30 days. This replaces email for document exchange — keeps conversation complete inside the app.

Interview Card

Special message type that renders as a "Schedule Interview" card with time slot picker. Both parties confirm, calendar invite auto-generated. Reminders sent 24h and 1h before. Outcome tagging post-interview.

Read Receipts & Typing Indicators

Standard messaging UX. Delivered / Read tick marks. Typing indicator via WebSocket presence event. Message search within a conversation. Optional: employer can mark conversation as "high priority" for internal tracking.

Unmatch & Conversation Archive

Either party can unmatch — removes the conversation but the match event is preserved in backend for analytics. Conversations auto-archived after 90 days of inactivity with option to re-open. Deleted messages logged (not content, just events) for safety audit trail.

LinkedIn Exchange Prompt

After 5+ messages, app shows a gentle prompt: "Ready to connect on LinkedIn?" Both users can share LinkedIn URLs in-app (pre-filled from their profile). This is the natural exit ramp — but the app has captured the full match-to-conversation funnel data.

Tech Stack

Production-grade, scalable, defensible technology choices

Mobile Apps
React Native + Expo

Single codebase for iOS and Android. Expo EAS for OTA updates (push fixes without App Store review). React Native Reanimated 3 for butter-smooth swipe animations (runs on UI thread, 60fps guaranteed). Gesture Handler for swipe detection.

State: Zustand + React Query

Zustand for lightweight client state (user session, swipe deck buffer). React Query for server state with automatic background refetch, optimistic UI updates on swipes.

Web App
Next.js 14 (App Router)

SSR for SEO on public pages (company pages, job posts). Client-side rendering for swipe deck (needs JS animation). Edge runtime for auth middleware. Deployed on Vercel for automatic scaling.

TailwindCSS + shadcn/ui

Fast, consistent UI development. shadcn for accessible component primitives. Custom design tokens for brand colors. Framer Motion for web-side swipe animations.

Backend API
Node.js + Express / Fastify

REST API for CRUD operations. Fastify preferred for higher throughput (benchmarks 2x Express on swipe-heavy endpoints). tRPC optional for type-safe API between Next.js web and backend.

Socket.io (WebSocket layer)

Real-time chat, typing indicators, match notifications, read receipts. Redis Adapter for Socket.io to support horizontal scaling across multiple Node.js instances.

AI / ML Service
Python + FastAPI

Separate ML microservice. Handles: compatibility score computation, Elo rating updates, skill vector embeddings (using sentence-transformers), AI Profile Coach (GPT-4o), content moderation. Deployed on AWS Lambda or a small EC2 for cost efficiency early on.

pgvector + Postgres

Store skill embeddings as vectors in Postgres via pgvector extension. Nearest-neighbour search for skill similarity without needing a separate vector DB. Simpler architecture, fewer moving parts at early scale.

Database & Storage
PostgreSQL (primary)

Users, profiles, swipes, matches, messages, reports. JSONB for flexible skill tags and filter preferences. pgvector for embeddings. Hosted on AWS RDS or Supabase. Connection pooling via PgBouncer at scale.

Redis (cache + queues)

Swipe deck pre-fetching, session storage, real-time match detection pub/sub, rate limiting (swipes/day counter), Socket.io adapter, boost queue, block list. AWS ElastiCache or Upstash (serverless Redis, cheaper at early stage).

Infrastructure & DevOps
AWS (primary cloud)

EC2 / ECS for API, RDS for Postgres, S3 for media, CloudFront CDN, SES for emails, SNS + FCM/APNs for push notifications, Rekognition for image moderation, Textract for document verification.

Docker + GitHub Actions CI/CD

Containerised services. GitHub Actions for automated test + deploy pipeline. Environment parity: dev / staging / prod. Sentry for error tracking. DataDog or Grafana for APM and infrastructure monitoring.

IP & Competitive Moat Strategy

How to make Mutch impossible to copy

The core insight: apps are easy to copy. Data and network effects are not.

Anyone can build a swipe UI. The moat is in the proprietary data — billions of swipe signals, compatibility outcomes, and behavioural patterns — that make the matching engine better than any new entrant can possibly replicate. Build aggressively toward data density from Day 1.

1. Proprietary Compatibility Algorithm

The scoring model is trained on your swipe + outcome data. Every hire that happens through the platform feeds back into the model. After 100K matches, your algorithm will outperform any competitor starting fresh — and the gap widens exponentially. Keep the model weights and feature engineering completely private. Do not describe it in job listings. Do not publish papers about it.

2. Network Density as a Moat

A matching app with 10K users is useless. With 1M users in a city, it becomes irreplaceable. Go city-deep before going wide. Dominate one geography's hiring ecosystem before expanding. Lock in the top companies first (employer side), then talent follows. This is the LinkedIn playbook — applied with a swipe mechanic.

3. Employer Ratings (Exclusive Data)

Mutch-native employer ratings — collected from actual hiring conversations and interviews — will be more authentic and more recent than Glassdoor data (which is often gamed). This data asset becomes irreplaceable. No competitor can replicate 2 years of verified post-interview ratings without starting from scratch.

4. Patent the Mutual-Intent Hiring Flow

File a provisional patent on the specific combination of: mutual swipe-based intent signalling + professional profile matching + in-app interview scheduling + outcome feedback loop. The individual components aren't patentable — the specific combination and application to professional hiring may be. Consult an IP attorney in India and the US simultaneously.

5. Skill Taxonomy (Proprietary Ontology)

Build a proprietary skill graph: skills linked to roles, linked to companies, linked to compensation bands. This is your knowledge graph. LinkedIn has one; you need your own. After 500K profiles, your skill-to-salary mapping will be more current and India-specific than any other source. License it to companies as a separate data product.

6. Contractual Lock-in (Enterprise)

When an enterprise client integrates Mutch via API with their ATS, switching costs become high. Their pipeline data lives in Mutch. Their recruiters are trained on it. Annual contracts with data portability terms that favour you legally. Enterprise ATS integration is a moat, not just a revenue line.

Anti-Fatigue Rules (critical for retention)

Maximum 3 push notifications per user per day. Never send push between 11pm–7am. Email digest max 1 per week unless transactional. Users can set per-category preferences in settings. Users who opt out of marketing email still get transactional (match, interview). A/B test copy continuously — "You have a new match" vs "Someone is interested in working with you" can differ 40% in CTR.

Launch Roadmap

From MVP to market dominance

Month 1–2
Build & Closed Beta

Build Phase 1 (auth, onboarding, swipe, match, basic chat). Invite 200–500 beta users: 50% employers, 50% employees. Target: tech industry in one city (Bangalore or Delhi NCR). Goal: measure match rate, message rate, time-to-match. Fix UX issues before public launch.

Month 3–4
Public Launch (City 1)

Launch publicly in target city. PR push: "Tinder for Jobs launches in Bangalore." Partner with 10 startups as design partners (free premium in exchange for feedback and public endorsement). Target 10K users in 60 days. Add filters and verification in this phase.

Month 5–6
Monetisation Launch

Turn on premium tiers. Target: 3–5% conversion to paid. Launch Profile Insights to drive employee premium. Launch "Who Liked You" for employers. Begin collecting employer ratings. Start building AI compatibility model on accumulated swipe data.

Month 7–12
Geographic & Vertical Expansion

Expand to 3 more cities. Add second vertical (e.g. design, finance, marketing — don't be only-tech). Launch freelance mode. Begin enterprise API conversations with staffing firms. Target: 100K users, ₹10–20L MRR, Series A readiness.
